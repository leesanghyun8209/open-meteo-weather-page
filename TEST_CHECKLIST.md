# TEST_CHECKLIST

## 설치와 실행

- [ ] `npm install`이 성공한다.
- [ ] `npm run test`가 성공한다.
- [ ] `npm run typecheck`가 성공한다.
- [ ] `npm run build`가 성공한다.
- [ ] `npm run dev` 실행 후 브라우저에서 `http://localhost:3000`에 접속할 수 있다.

## 지역 선택 GUI

- [ ] 최초 진입 시 브라우저 위치 권한 요청이 표시된다.
- [ ] 위치 권한을 허용하면 지도 기본 핀이 사용자 위치로 이동한다.
- [ ] 위치 권한을 허용하면 `Current location` 대신 실제 지역명이 표시된다.
- [ ] 위치 권한을 거부하면 서울이 기본 지역으로 설정되고 안내 문구가 표시된다.
- [ ] 지도 클릭 시 핀이 클릭한 좌표로 이동하고 선택 위치 텍스트가 실제 지역명으로 갱신된다.
- [ ] 지역명을 찾지 못하는 좌표는 좌표 기준 이름과 친절한 안내를 표시한다.
- [ ] 도시 검색어 입력 후 검색하면 Open-Meteo Geocoding 결과가 표시된다.
- [ ] 검색 결과를 클릭하면 지도 핀과 선택 위치가 해당 지역으로 변경된다.

## 날짜와 Open-Meteo 제약

- [ ] `오늘` 탭에서는 오늘 날짜가 고정되고 현재 날씨와 일별 요약이 표시된다.
- [ ] `미래` 탭에서는 내일부터 오늘 포함 16일 예보 범위까지만 선택 가능하다.
- [ ] 미래 범위를 초과한 날짜는 UI에서 선택할 수 없고 서버에서도 거부된다.
- [ ] `과거` 탭에서는 `1940-01-01` 이후, 어제 이전 날짜만 선택 가능하다.
- [ ] `1940-01-01` 이전 날짜 요청은 서버에서 친절한 오류로 거부된다.
- [ ] 과거/미래 응답에 선택 날짜 데이터가 없으면 “선택한 날짜의 관측/재분석 데이터가 아직 제공되지 않습니다” 안내가 표시된다.

## 날씨 조회 결과

- [ ] 날씨 조회 성공 시 위치명, 조회 날짜, API 소스, 날씨 상태가 표시된다.
- [ ] 오늘 조회는 현재 기온, 체감 온도, 습도를 표시한다.
- [ ] 과거/미래 조회는 일별 최고/최저 기온, 강수량, 풍속을 표시한다.
- [ ] 네트워크 오류나 Open-Meteo 오류가 발생하면 사용자가 이해할 수 있는 오류 메시지가 표시된다.

## SQLite 저장과 최근 조회

- [ ] 성공한 조회만 `data/weather.sqlite`에 저장된다.
- [ ] `weather_queries` 테이블에 조회 모드, 위치 소스, 좌표, 요청 날짜, API 소스, 날씨 값, 원본 응답이 저장된다.
- [ ] 최근 조회 영역은 최신순 10건만 표시한다.
- [ ] 최근 조회 항목을 클릭하면 해당 좌표, 날짜, 모드로 다시 조회된다.
- [ ] 브라우저를 새로고침해도 SQLite에 저장된 최근 조회가 유지된다.

## 반응형 화면

- [ ] 데스크톱에서 지도, 검색, 날짜 선택, 결과, 최근 조회가 겹치지 않는다.
- [ ] 모바일 폭에서 버튼 텍스트와 카드 내용이 부모 영역 밖으로 넘치지 않는다.
- [ ] 지도 높이가 모바일에서도 충분히 유지되고 핀 위치가 보인다.
